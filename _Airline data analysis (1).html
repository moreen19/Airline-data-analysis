US AIRLINE DATA ANALYSIS¶

Author:Moreen Owino

This data analysis is done to aid an airline company looking to enter the United States domestic market which has identified medium and large airports as their desired operating locations. The company believes that it has a competitive advantage in maintaining punctuality, so it plans on making this a big part of its brand image with a motto, “On time, for you.” To kick start operations, the company has decided to start with 5 round trip routes.

The analysis aims to analyze 1Q2000 data and identify:

  1. The 6 busiest round trip routes in terms of number of round trip flights in the quarter. Excluding canceled flights.

  2. The 6 most profitable round trip routes (without considering the upfront airplane cost) in the quarter. Along with the profit, showing total revenue, total cost, summary values of other key components and total round trip flights in the quarter for the top 6 most profitable routes. Excluding canceled flights.

  3. The 5 round trip routes that i recommend to invest in based on any factors that i choose.

  4. The number of round trip flights it will take to breakeven on the upfront airplane cost for each of the 5 round trip routes that you recommend. showing key summary components for these routes.

  5. Key Performance Indicators (KPI’s) that i recommend tracking in the future to measure the success of the round trip routes that i recommend.

  6. My conclusion and recommendations.

# creating function to remove and detect outliers

def remove_outliers_iqr(dataframe, column_name, lower_threshold, upper_threshold):
    """
    Remove outliers from a DataFrame column using the IQR method.

    Parameters:
    - dataframe: The DataFrame containing the data.
    - column_name: The name of the column with outliers.
    - threshold: The IQR threshold for identifying outliers. Default is 1.5.

    Returns:
    - A new DataFrame with outliers removed.
    """
    Q1 = dataframe[column_name].quantile(0.25)
    Q3 = dataframe[column_name].quantile(0.75)
    IQR = Q3 - Q1
    lower_bound = Q1 - lower_threshold * IQR
    upper_bound = Q3 + upper_threshold * IQR
    return dataframe[(dataframe[column_name] >= lower_bound) & (dataframe[column_name] <= upper_bound)]

# Remove outliers in the 'ITIN_FARE' column using the function
# the itin_fare seems to have some pretty low values and some very high values too as seen in the output from describe func

# creating function
def check_for_duplicates(data, dataset_name):
    """
    Check for duplicates in a given dataset, print the first 5 rows of duplicate entries, and remove duplicates.

    Parameters:
        data (pd.DataFrame): The dataset to check for duplicates.
        dataset_name (str): A string specifying the name of the dataset.

    Returns:
        pd.DataFrame: A DataFrame with duplicates removed.

    Example:
        deduplicated_data = check_for_duplicates(flights_data, "Flights dataset")
        print("Duplicate entries in Flights dataset:")
    """
    duplicate_entries = data[data.duplicated()]
    if not duplicate_entries.empty:
        print(f"Duplicate entries in {dataset_name} ({len(duplicate_entries)} entries):")
        print("\n")
        print(duplicate_entries.head())  # Print the first 5 rows of duplicates
        data = data.drop_duplicates()  # Remove duplicates from the original dataset
    else:
        print(f"No duplicate entries found in {dataset_name}.")
    return data

Insights on data quality¶

  • There data contained a lot duplicates especially the tickets data set that had 68,233 duplicate entries
  • The data had data stored in the wrong data types like having passangers recorded as type float etc
  • The data sets were incomplete and had columns that had missing values e.g the IATA column in airportcodes data set with 46187
  • The data also had some outliers in key columns such as ITIN_FARE and DISTANCE

Hence the data sets had quality issues that needed to be addressed and have been handled as documented.

Data Munging¶

First we will merge the data sets inorder to have a data set that can help gain insight on the analysis questions that we have. and we'll do that by first creating a function called merge_and_filter_datasets that we can reuse later for other merging tasks.

def merge_and_filter_datasets(left_df, right_df, left_merge_columns, right_merge_columns, left_columns, right_columns, how):
    """
    Merge and filter two datasets based on specified columns or all columns as needed.

    Parameters:
    left_df (pd.DataFrame): Left dataset.
    right_df (pd.DataFrame): Right dataset.
    left_merge_columns (list of str): List of columns from the left dataset to merge on.
    right_merge_columns (list of str): List of columns from the right dataset to merge on.
    left_columns (list of str): List of columns to select from the left dataset.
    right_columns (list of str): List of columns to select from the right dataset.
    how (str): A string indicating how you want to merge, e.g., "left", "right", "inner"

    Returns:
    pd.DataFrame: Merged dataset with selected columns.
    """
     # If left_columns is None, include all columns from the left dataset
    if left_columns is None:
        left_columns = list(left_df.columns)
        
    # If left_columns is None, include all columns from the left dataset
    if right_columns is None:
        right_columns = list(right_columns)
    
    
    # Filter relevant columns from the left dataset
    left_df = left_df[left_columns]

    # Filter relevant columns from the right dataset
    right_df = right_df[right_columns]

    # Merge the datasets based on specified merge columns and how parameter
    merged_df = pd.merge(left_df, right_df, how=how, left_on=left_merge_columns, right_on=right_merge_columns)

    return merged_df

5. Recommended Key Performance Indicators (KPI’s) to track in the future to measure the success of the round-trip routes recommended.

To measure the success of the recommended round-trip routes, you should track several Key Performance Indicators (KPIs). Here are some KPIs that I recommend monitoring:

Profit Margin: Calculate the profit margin for each route, which is the percentage of profit relative to total revenue. A higher profit margin indicates a more successful route.

Occupancy Rate / load factor: Continuously track the average occupancy rate of flights on each route. A higher occupancy rate means that more seats are filled, leading to increased revenue.

Flight Punctuality: Monitor on-time performance, specifically the percentage of flights that depart and arrive on time. Higher punctuality enhances customer satisfaction and may attract more passengers.

Customer Satisfaction: Collect feedback from passengers to measure their satisfaction with the service. Happy customers are more likely to become repeat customers and refer others.

Marketing and Promotion: Evaluate the effectiveness of marketing and promotional campaigns for each route. Track the return on investment (ROI) for these activities.

Route-Specific Costs: Continuously assess the costs specific to each route, including fuel, maintenance, crew, and airport fees. Identify cost-saving opportunities.

Market Demand: Keep an eye on changes in market demand. If there's a shift in passenger preferences or if new competitors enter the market, adapt your strategy accordingly.

Price: Adjust pricing strategies based on demand and competition. Ensure that fare structures are optimized to maximize revenue.

Competition: Regularly analyze the competitive landscape on each route. Monitor the actions of competitors and respond accordingly.

Operational Efficiency: Monitor operational efficiency, such as aircraft turnaround time, to minimize delays and reduce costs.

Customer Acquisition Costs: Calculate the cost of acquiring each customer on a specific route. This can help assess the effectiveness of marketing efforts.

Financial Performance: Keep a close eye on financial performance, including cash flow, return on investment, and net profit, for each route.

These KPIs will provide an overview of the success of the recommended round-trip routes and help make informed decisions to improve profitability and customer satisfaction. Regularly reviewing and analyzing these metrics will be crucial for route management and decision-making.
